# SSH Commander

Десктопное приложение для управления SSH-подключениями с поддержкой вкладок, файлового менеджера и шифрованного хранилища паролей.

## Возможности

### Терминал
- Подключение к SSH-серверам с полноценным терминалом (PTY)
- Несколько сессий во вкладках одновременно
- Быстрое подключение (`Ctrl+K`) без сохранения
- Группировка подключений по категориям

### Файловый менеджер
- Двухпанельный Commander-стиль (локальная ФС и/или SFTP)
- Копирование файлов: локально, upload, download, сервер-сервер
- Прямой SCP между серверами (через sshpass) с автоматическим fallback на потоковый транзит
- Докачка файлов при разрыве соединения (resume)
- Автоматические повторные попытки с экспоненциальным backoff (до 3 попыток)
- Рекурсивное копирование директорий
- Drag & Drop между панелями
- Подтверждение перезаписи существующих файлов
- Фильтр по имени (мгновенный)
- Рекурсивный поиск по glob-маскам (`*.py`, `*.log`, `config*`)
- Изменяемая ширина колонок

### Менеджер баз данных
- Подключение к PostgreSQL, MySQL, SQLite через SSH-туннель
- Обзор баз данных и таблиц (дерево)
- Просмотр данных с пагинацией (100 строк/стр) и сортировкой
- Inline-редактирование ячеек (UPDATE по первичному ключу)
- SQL-редактор с выполнением запросов (Ctrl+Enter)
- Экспорт результатов в CSV и JSON

### Заметки к серверам
- Markdown-заметки привязанные к SSH-подключениям
- Редактирование прямо в диалоге подключения
- Экспорт/импорт вместе с подключениями

### Массовое выполнение команд
- Выбор нескольких серверов (чекбоксы)
- Параллельное выполнение команды на всех выбранных
- Таблица результатов: сервер, статус, вывод, время выполнения
- Экспорт результатов в текст

### Docker-менеджер
- Список контейнеров через `docker ps --format json`
- Start / Stop / Restart контейнеров
- Просмотр логов (docker logs)
- Inspect контейнера (JSON)
- Выполнение команд внутри контейнера (docker exec)

### Сетевые утилиты
- **Ping**: хост, кол-во пакетов, локально или через SSH
- **Traceroute**: fallback traceroute -> tracepath -> mtr
- **DNS Lookup**: dig / nslookup, выбор типа записи (A, AAAA, MX, NS, TXT, CNAME, SOA, PTR)
- **Сканер портов**: одиночный или диапазон, прогресс-бар
- Кнопка остановки для всех утилит
- Выполнение локально или на удалённом сервере

### Сравнение файлов
- Двухпанельный diff-viewer (Unified Diff)
- Источники: локальный файл или файл с SSH-сервера
- Подсветка изменений: зелёный (добавлено), красный (удалено)
- Обмен панелей местами
- Подсказки для SSH-путей

### Системная информация
- Snapshot сервера одной кнопкой: OS, hostname, ядро, CPU, RAM, диски, IP, uptime
- Прогресс-бары для RAM и дисков с цветовой индикацией (зелёный/жёлтый/красный)
- Сбор данных одной SSH-командой

### SSH-туннели
- GUI для Local (-L) и Remote (-R) port forwarding
- Таблица активных туннелей с кнопкой отключения
- Поддержка нескольких туннелей одновременно
- Корректное закрытие при выходе из приложения

### Systemd-менеджер
- Список сервисов с фильтрацией по имени и состоянию
- Start / Stop / Restart / Enable / Disable сервисов
- Просмотр логов через journalctl с фильтром по приоритету
- Цветовая индикация: active (зелёный), failed (красный), inactive (серый)

### Избранные команды
- Сохранение часто используемых команд
- Выполнение в активном терминале одним кликом
- Категоризация и описание команд

### Менеджер секретов
- Хранение паролей, токенов, ключей API
- Категоризация записей
- Шифрование паролей мастер-ключом
- Копирование логина/пароля в буфер

### Безопасность
- Мастер-пароль для доступа к приложению
- Шифрование паролей подключений (Fernet/PBKDF2)
- Блокировка после 3 неверных попыток
- Смена мастер-пароля с перешифровкой всех данных

### Импорт/экспорт
- **Экспорт SSH**: отдельные подключения в JSON (с паролями и подключениями к БД)
- **Импорт SSH**: из JSON с проверкой дублей и диалогом замены
- **Глобальный экспорт**: все данные приложения (SSH, БД, секреты, заметки) в JSON
- **Глобальный импорт**: восстановление из JSON с дедупликацией всех сущностей

## Установка и запуск

### Требования

- Python 3.10+
- Ubuntu / Linux (тестировалось на Ubuntu 22.04+)

### Из исходников (для разработки)

```bash
# Клонирование
git clone <repository_url>
cd cmd

# Виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate

# Зависимости
pip install -r requirements.txt

# Запуск
make run
```

### Сборка бинарника

```bash
# Установка dev-зависимостей
pip install -r requirements.dev.txt

# Сборка (результат в dist/ssh-commander, ~74MB)
make build

# Установка в систему (бинарник + .desktop файл)
make install

# Удаление из системы
make uninstall
```

После `make install` приложение появится в меню Ubuntu.

## Хранение данных

Все данные хранятся в `~/.local/share/ssh-commander/`:

| Файл | Описание |
|------|----------|
| `connections.db` | SQLite база (подключения, команды, БД, секреты, заметки) |
| `crypto.key` | Соль + зашифрованный verify-токен |

Схема БД: v4 (таблицы: connections, commands, db_connections, secrets, server_notes).

Пароли подключений шифруются Fernet с ключом, выведенным из мастер-пароля через PBKDF2 (100 000 итераций).

### Основные
- **PySide6** >= 6.6.0 - Qt6 GUI фреймворк
- **paramiko** >= 3.4.0 - SSH/SFTP клиент
- **pyte** >= 0.8.2 - эмуляция VT100 терминала
- **cryptography** >= 42.0.0 - шифрование паролей
- **psycopg2-binary** >= 2.9.0 - PostgreSQL клиент
- **PyMySQL** >= 1.1.0 - MySQL/MariaDB клиент

### Разработка
- **pyinstaller** >= 6.0.0 - сборка бинарника
- **pytest** / **pytest-qt** - тесты
- **ruff** - линтер
- **mypy** - проверка типов

## Makefile команды

```bash
make run          # Запуск из исходников
make run-noauth   # Запуск без мастер-пароля
make reset        # Сброс всех данных (БД + ключи)
make build        # Сборка бинарника (PyInstaller)
make install      # Установка в систему
make uninstall    # Удаление из системы
make lint         # Проверка кода (ruff)
make test         # Запуск тестов
make clean        # Очистка артефактов сборки
```

## CLI-аргументы

```bash
ssh-commander              # Обычный запуск (с мастер-паролем)
ssh-commander --no-auth    # Без мастер-пароля
ssh-commander --reset      # Сброс данных (с подтверждением)
```

Для бинарника: `./dist/ssh-commander --no-auth`

## Известные ограничения

- При уменьшении окна текст в терминале может обрезаться (pyte не поддерживает reflow)
- Рекурсивный поиск по SFTP ограничен 500 результатами (защита от перегрузки)
- Docker-менеджер требует установленного Docker на целевом сервере
- Traceroute: если нет утилиты, предлагает установить через sudo apt install
