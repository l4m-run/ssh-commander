# SSH Commander

Десктопное приложение для управления SSH-подключениями с поддержкой вкладок, файлового менеджера и шифрованного хранилища паролей.

## Возможности

### Терминал
- Подключение к SSH-серверам с полноценным терминалом (PTY)
- Несколько сессий во вкладках одновременно
- Быстрое подключение (`Ctrl+K`) без сохранения
- Группировка подключений по категориям

### Файловый менеджер
- Двухпанельный Commander-стиль (локальная ФС и/или SFTP)
- Копирование файлов: локально, upload, download, сервер-сервер
- Прямой SCP между серверами (через sshpass) с автоматическим fallback на потоковый транзит
- Докачка файлов при разрыве соединения (resume)
- Автоматические повторные попытки с экспоненциальным backoff (до 3 попыток)
- Рекурсивное копирование директорий
- Drag & Drop между панелями
- Подтверждение перезаписи существующих файлов
- Фильтр по имени (мгновенный)
- Рекурсивный поиск по glob-маскам (`*.py`, `*.log`, `config*`)
- Изменяемая ширина колонок

### Избранные команды
- Сохранение часто используемых команд
- Выполнение в активном терминале одним кликом
- Категоризация и описание команд

### Безопасность
- Мастер-пароль для доступа к приложению
- Шифрование паролей подключений (Fernet/PBKDF2)
- Блокировка после 3 неверных попыток
- Смена мастер-пароля с перешифровкой всех данных

### Импорт/экспорт
- Экспорт подключений в JSON (опционально с паролями)
- Импорт из JSON с проверкой дублей и поддержкой паролей

## Установка и запуск

### Требования

- Python 3.10+
- Ubuntu / Linux (тестировалось на Ubuntu 22.04+)

### Из исходников (для разработки)

```bash
# Клонирование
git clone <repository_url>
cd cmd

# Виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate

# Зависимости
pip install -r requirements.txt

# Запуск
make run
```

### Сборка бинарника

```bash
# Установка dev-зависимостей
pip install -r requirements.dev.txt

# Сборка (результат в dist/ssh-commander, ~74MB)
make build

# Установка в систему (бинарник + .desktop файл)
make install

# Удаление из системы
make uninstall
```

После `make install` приложение появится в меню Ubuntu.

## Горячие клавиши

|---------|----------|
| `Ctrl+N` | Новое подключение |
| `Ctrl+K` | Быстрое подключение |

## Хранение данных

Все данные хранятся в `~/.local/share/ssh-commander/`:

| Файл | Описание |
|------|----------|
| `connections.db` | SQLite база (подключения, команды) |
| `crypto.key` | Соль + зашифрованный verify-токен |

Пароли подключений шифруются Fernet с ключом, выведенным из мастер-пароля через PBKDF2 (100 000 итераций).

## Зависимости

### Основные
- **PySide6** >= 6.6.0 - Qt6 GUI фреймворк
- **paramiko** >= 3.4.0 - SSH/SFTP клиент
- **pyte** >= 0.8.2 - эмуляция VT100 терминала
- **cryptography** >= 42.0.0 - шифрование паролей

### Разработка
- **pyinstaller** >= 6.0.0 - сборка бинарника
- **pytest** / **pytest-qt** - тесты
- **ruff** - линтер
- **mypy** - проверка типов

## Makefile команды

```bash
make run          # Запуск из исходников
make run-noauth   # Запуск без мастер-пароля
make reset        # Сброс всех данных (БД + ключи)
make build        # Сборка бинарника (PyInstaller)
make install      # Установка в систему
make uninstall    # Удаление из системы
make lint         # Проверка кода (ruff)
make test         # Запуск тестов
make clean        # Очистка артефактов сборки
```

## CLI-аргументы

```bash
ssh-commander              # Обычный запуск (с мастер-паролем)
ssh-commander --no-auth    # Без мастер-пароля
ssh-commander --reset      # Сброс данных (с подтверждением)
```

Для бинарника: `./dist/ssh-commander --no-auth`

## Известные ограничения

- При уменьшении окна текст в терминале может обрезаться (pyte не поддерживает reflow)
- Рекурсивный поиск по SFTP ограничен 500 результатами (защита от перегрузки)
